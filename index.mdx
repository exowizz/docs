Exo is a Laravel package that turns your Eloquent models into an automation-ready API. Define which models to expose, what data to share, and which events to listen for. Exo handles the rest — API endpoints, webhook delivery, OAuth authentication, and integration with platforms like Zapier, Make, and n8n.

## How it works

1. **You define resources** — Pick your Eloquent models (orders, contacts, invoices) and wrap them in a resource class
2. **Exo creates the API** — A REST API appears at `/exo-api` with list, create, update, and delete endpoints
3. **Webhooks fire automatically** — When a model is created, updated, or deleted, Exo notifies any subscribed webhook URLs
4. **Platforms connect** — Push your schema to Exo Cloud and your app shows up in Zapier, Make, or n8n

## Key features

- **Resource classes** — Map Eloquent models to API endpoints with transforms, validation, and ownership rules
- **Automatic webhooks** — Subscribe to `on_create`, `on_update`, and `on_delete` events per resource
- **Built-in authentication** — OAuth2 via Laravel Passport with personal access tokens and authorization flows
- **Web dashboard** — Manage webhook subscriptions and API tokens through a browser UI
- **CLI tools** — Scaffold resources, create users, generate schemas, and deploy to Exo Cloud from the terminal
- **Ownership & authorization** — Scope data to individual users or grant full access to admins
